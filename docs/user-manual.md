## 1. ภาพรวมระบบ

ระบบ **JP Stock Packing** เป็นระบบบริหารจัดการสต็อกและการบรรจุสินค้า สำหรับโรงงานผลิตเครื่องประดับ ครอบคลุมกระบวนการตั้งแต่รับคำสั่งผลิต ไปจนถึงการจัดส่งสินค้าไปยังปลายทางต่าง ๆ

### กระบวนการหลัก

```
รับคำสั่งผลิต → รับสินค้า → มอบหมายงาน → บรรจุสินค้า → จัดส่งปลายทาง
     ↓                                                           ↓
แผนการผลิต                                         ร้านค้า / หลอม / ส่งออก
                                                    โชว์รูม / สูญหาย
```

### ปลายทางการจัดส่ง

| ปลายทาง | ความหมาย |
|---------|---------|
| **Store** | จัดเข้าสต็อกร้านค้าปกติ |
| **Melt** | ส่งหลอม (นำกลับมาผลิตใหม่) |
| **Export** | ส่งออกต่างประเทศ |
| **Showroom** | ส่งโชว์รูม |
| **Lost** | สินค้าสูญหาย/เสียหาย (ตัดบัญชี) |

## 2. การเข้าสู่ระบบ

### ขั้นตอนการเข้าสู่ระบบ

1. เปิดเบราว์เซอร์และเข้าไปที่ URL ของระบบ
2. กรอก **ชื่อผู้ใช้** และ **รหัสผ่าน**
3. (ไม่บังคับ) ติ๊ก **"จดจำฉัน"** เพื่อให้ระบบจำการเข้าสู่ระบบไว้ 7 วัน
4. กดปุ่ม **เข้าสู่ระบบ**

### การออกจากระบบ

- คลิกที่ไอคอนหรือชื่อผู้ใช้งานแล้วเลือก **ออกจากระบบ**
- ระบบจะล้างข้อมูล session และคุกกี้ทั้งหมด

> **หมายเหตุ**: หากระบบไม่ตอบสนองหรือหมดอายุ session ระบบจะแสดงหน้าเข้าสู่ระบบอัตโนมัติ

## 4. การจัดการคำสั่งผลิต (Order Management)

### การดูรายการคำสั่งผลิต

1. คลิก **Order Management** ในเมนูด้านซ้าย
2. ระบบแสดงรายการคำสั่งผลิตพร้อมข้อมูลล็อต

### การค้นหาและกรองข้อมูล

สามารถกรองได้ด้วย:
- **Order No** – เลขที่คำสั่งผลิต
- **Lot No** – เลขที่ล็อต
- **Customer Code** – รหัสลูกค้า
- **ช่วงวันที่** – วันเริ่มต้นและสิ้นสุด

กดปุ่ม **ค้นหา** เพื่อกรองผลลัพธ์ ระบบรองรับการแบ่งหน้า (Pagination)

### การดูรายละเอียดล็อต

- คลิกที่แถวของล็อตเพื่อดูสินค้าที่รับเข้ามาในล็อตนั้น
- ดูจำนวนและสถานะสินค้าแต่ละรายการ

### การนำเข้าใบรับสินค้า

1. เลือกล็อตที่ต้องการ
2. กดปุ่ม **รับเข้า**
3. ระบบจะดึงข้อมูลจากระบบ JP
4. ยืนยันการนำเข้า

### การยกเลิกการนำเข้า

- กดปุ่ม **ยกเลิกการนำเข้า** เพื่อยกเลิกรายการที่นำเข้ามา

---

## 5. แผนการบรรจุ (Packing Planning)

### การดูแผนการผลิต

1. คลิก **Packing Planning** ในเมนู
2. ระบบแสดงคำสั่งผลิตที่รอวางแผน พร้อมข้อมูล:
   - จำนวนชิ้น
   - ประเภทสินค้า
   - จำนวนวันทำงานที่คำนวณอัตโนมัติ

### การคำนวณวันทำงาน

ระบบคำนวณวันทำงานของแต่ละล็อตโดยอัตโนมัติ โดยอิงจาก:
- **จำนวนชิ้น** ในล็อต
- **ประเภทสินค้า** (แต่ละประเภทมี Formula การคำนวณต่างกัน)

### การจัดกลุ่มลูกค้า

- ระบบจัดกลุ่มคำสั่งผลิตตามลูกค้าเพื่อให้วางแผนได้สะดวก

---

## 6. การรับสินค้า (Receive Management)

เมนูนี้ใช้สำหรับรับสินค้าตามใบแจ้งหนี้ (SJ1 = สต็อกร้าน, SJ2 = หลอม)

### ขั้นตอนการรับสินค้า

1. คลิก **Receive Management**
2. ค้นหาใบรับสินค้าที่ต้องการ
3. คลิกเลือกรายการ
4. กด **Import** เพื่อยืนยันการรับสินค้า
5. ระบบบันทึกสถานะและอัปเดตข้อมูล

### การยืนยันรายการล็อต

1. เลือกล็อตที่รับแล้ว
2. กด **Update Lot Items** เพื่อยืนยันรายการ
3. หากต้องการแก้ไข กด **Cancel Update** เพื่อย้อนสถานะ

### การอัปเดตตาม Revision Number

- กด **Update by Rev No** เพื่ออัปเดตรายการตามหมายเลข Revision

---

## 7. การรับตัวอย่างสินค้า (Sample Receive)

ใช้กระบวนการเดียวกับ Receive Management แต่สำหรับสินค้าตัวอย่าง (Sample)

### ขั้นตอน

1. คลิก **Sample Receive Management**
2. ค้นหาและเลือกรายการตัวอย่าง
3. กด **Import Sample** เพื่อรับ
4. กด **Update Sample Lot Items** เพื่อยืนยัน
5. หากต้องการยกเลิก กด **Cancel Update**

---

## 8. การมอบหมายงาน (Assignment)

### การมอบหมายสินค้าให้โต๊ะทำงาน

1. ใน Order Management เลือกล็อตที่ต้องการมอบหมาย
2. กด **Assign to Table**
3. เลือก **โต๊ะทำงาน** (Table) ที่จะมอบหมาย
4. ระบบแสดงสมาชิกในทีม
5. ยืนยันการมอบหมาย

### การดูสมาชิกทีม

- กด **Get Table Member** เพื่อดูรายชื่อสมาชิกในโต๊ะนั้น ๆ
- หรือค้นหาตาม Assignment ID

### การคืนสินค้าจากการมอบหมาย

1. เลือกโต๊ะที่ต้องการคืนสินค้า
2. กด **Return Assignment**
3. ระบุจำนวนที่คืน
4. ยืนยันการคืน ระบบจะติดตามจำนวนที่คืนไว้

---

## 9. การบรรจุและจัดส่งสินค้า (Packed Management)

เมนูหลักสำหรับจัดการสินค้าที่บรรจุเสร็จแล้วและส่งไปยังปลายทาง

### การดูสินค้าพร้อมส่ง

1. คลิก **Packed Management**
2. ระบบแสดงรายการสินค้าที่พร้อมจัดส่ง

### การส่งสินค้าไปปลายทาง

#### ส่งร้านค้า (Send to Store)
1. เลือกรายการสินค้า
2. กด **Send to Store**
3. ระบุจำนวนและล็อตที่ส่ง
4. กด **Confirm** เพื่อยืนยัน

#### ส่งหลอม (Send to Melt)
1. เลือกรายการสินค้า
2. กด **Confirm to Send Melt**
3. ยืนยันการส่ง

#### ส่งออก (Send to Export)
1. เลือกรายการ
2. กด **Confirm to Send Export**
3. ยืนยัน

#### ส่งโชว์รูม (Send to Showroom)
1. เลือกรายการ
2. กด **Confirm to Send Showroom**
3. ยืนยัน

#### บันทึกสูญหาย (Send to Lost)
1. เลือกรายการ
2. กด **Confirm to Send Lost**
3. ยืนยัน

#### ยืนยันทั้งหมดพร้อมกัน (Confirm All)
- กด **Confirm to Send All** เพื่อยืนยันการส่งทุกปลายทางพร้อมกัน

> **หมายเหตุ**: ระบบบันทึกชื่อผู้ที่กด Confirm ทุกครั้งเพื่อ Audit Trail

## 10. การตรวจสอบจำนวน (Check Qty to Pack)

ใช้ตรวจสอบจำนวนสินค้าก่อนบรรจุ โดยเปรียบเทียบกับจำนวนที่อนุมัติ

### ค่าความคลาดเคลื่อนที่ยอมรับได้

| ประเภท | ค่าเริ่มต้น |
|--------|----------|
| Send Qty Tolerance | 2% |
| Store Qty Tolerance | 5% |

### ขั้นตอน

1. คลิก **Check Qty to Pack**
2. ระบบแสดงรายการพร้อมจำนวนที่อนุมัติ
3. ตรวจสอบว่าจำนวนจริงอยู่ในช่วงที่ยอมรับได้
4. กด **Define to Pack** เพื่อกำหนดล็อตสำหรับบรรจุ

---

## 11. การบันทึกสินค้าชำรุด (Break)

### การบันทึกสินค้าชำรุด

1. ไปที่เมนูที่เกี่ยวข้อง (ผ่าน Validation หรือ Packed Management)
2. กด **Add Break**
3. เลือก **สาเหตุการชำรุด** (Break Description)
4. ระบุจำนวนชิ้นที่ชำรุด
5. บันทึก

### การเพิ่มหมวดหมู่สาเหตุใหม่

1. กด **Add New Break Description**
2. กรอกชื่อหมวดหมู่สาเหตุ
3. บันทึก

### การดูรายการสินค้าชำรุด

- กด **Get Break** เพื่อดูรายการพร้อมฟิลเตอร์ตามวันที่และล็อต

### การพิมพ์รายงานสินค้าชำรุด

- กด **Break Report** เพื่อสร้างรายงาน PDF

---

## 12. การบันทึกสินค้าสูญหาย (Lost)

### การบันทึกสินค้าสูญหาย

1. กด **Add Lost**
2. เลือก **หัวหน้าโต๊ะ** (Table Leader) ที่รับผิดชอบ
3. ระบุจำนวนและรายละเอียด
4. บันทึก

### การดูรายการสูญหาย

- กด **Get Lost** เพื่อดูรายการทั้งหมด

### การพิมพ์รายงานสินค้าสูญหาย

- กด **Lost Report** เพื่อสร้างรายงาน PDF

---

## 13. การตรวจสอบและ Audit

### การเปรียบเทียบใบแจ้งหนี้ (Compare Invoice)

1. ไปที่ **Validation**
2. ระบบแสดงการเปรียบเทียบ SJ1/SJ2 กับยอดที่รับ
3. ตรวจสอบความถูกต้องของตัวเลข
4. กด **Mark as Read** เพื่อยืนยันว่าตรวจสอบแล้ว

### การตรวจสอบสต็อกที่ยังไม่ได้จัดสรร (Unallocated Qty)

- ดูรายการสินค้าที่ยังไม่ถูกส่งไปปลายทางใด ๆ
- ใช้สำหรับติดตามสินค้าคงค้าง

### การตรวจสอบ Send Lost Checklist

- ดูรายการตรวจสอบก่อนบันทึกสินค้าสูญหาย
- ยืนยันความถูกต้องก่อนดำเนินการ

## 14. การยกเลิกรายการ (Cancel Receive)

ใช้สำหรับยกเลิกรายการที่ส่งหรือบันทึกไปแล้ว

### ประเภทการยกเลิก

| ประเภท | การยกเลิก |
|--------|---------|
| Store | ยกเลิกการส่งร้านค้า |
| Melt | ยกเลิกการส่งหลอม |
| Export | ยกเลิกการส่งออก |
| Showroom | ยกเลิกการส่งโชว์รูม |
| Lost | ยกเลิกการบันทึกสูญหาย |

### วิธีการยกเลิก

**ยกเลิกตาม Receive No:**
1. คลิก **Cancel Receive**
2. เลือกประเภทที่ต้องการยกเลิก (Store/Melt/Export/Showroom/Lost)
3. ค้นหาและเลือก Receive No
4. กด **Cancel by Receive No**
5. ยืนยันการยกเลิก

**ยกเลิกตาม Lot No:**
1. ทำขั้นตอนเดียวกัน แต่กด **Cancel by Lot No**
2. เลือก Lot ที่ต้องการยกเลิก
3. ยืนยัน

> **หมายเหตุ**: ระบบบันทึกชื่อผู้ยกเลิกทุกครั้ง

---

## 15. การพิมพ์รายงาน (Reports)

### รายงานที่มีในระบบ

| รายงาน | วิธีเข้าถึง |
|--------|----------|
| Qty to Pack Report | Packed Management → Print |
| Send to All Report | Packed Management → Print All |
| Send Report by Type | Packed Management → Print by Type |
| Break Report | บันทึกสินค้าชำรุด → Break Report |
| Lost Report | บันทึกสินค้าสูญหาย → Lost Report |
| Compared Invoice Report | Validation → Print |
| Unallocated Qty Report | Validation → Print Unallocated |

### การพิมพ์รายงาน

1. กดปุ่ม **Print** หรือ **Report** ตามเมนูที่ระบุ
2. ระบบสร้างไฟล์ PDF
3. เบราว์เซอร์จะดาวน์โหลดหรือเปิดไฟล์อัตโนมัติ

## 16. การจัดการผู้ใช้งาน (User Management)

> **สิทธิ์**: ต้องมีสิทธิ์ User Management จึงจะเข้าถึงได้

### การดูรายการผู้ใช้

1. คลิก **User Management**
2. ระบบแสดงรายชื่อผู้ใช้ทั้งหมด

### การเพิ่มผู้ใช้ใหม่

1. กด **Add New User**
2. กรอกข้อมูล:
   - เลือกพนักงาน (จากระบบ PIS)
   - กำหนดชื่อผู้ใช้และรหัสผ่าน
3. กด **บันทึก**

### การแก้ไขข้อมูลผู้ใช้

1. คลิกที่แถวผู้ใช้ที่ต้องการแก้ไข
2. กด **Edit**
3. แก้ไขข้อมูลที่ต้องการ
4. กด **บันทึก**

### การเปิด/ปิดบัญชีผู้ใช้

- คลิกที่ปุ่ม **Toggle Status** เพื่อเปิด/ปิดการใช้งานบัญชี
- บัญชีที่ปิดการใช้งานจะไม่สามารถเข้าสู่ระบบได้

## 18. การจัดการสิทธิ์การใช้งาน (Permission Management)

> **สิทธิ์**: ต้องมีสิทธิ์ Permission Management จึงจะเข้าถึงได้

### เมนูที่สามารถกำหนดสิทธิ์ได้

| เมนู | คำอธิบาย |
|------|---------|
| OrderManagement | จัดการคำสั่งผลิต |
| PackingPlaning | วางแผนการบรรจุ |
| ReceiveManagement | รับสินค้า |
| SampleReceiveManagement | รับตัวอย่างสินค้า |
| PackedManagement | จัดการสินค้าบรรจุแล้ว |
| CheckQtyToPack | ตรวจสอบจำนวน |
| Validation | ตรวจสอบ/Audit |
| CancelReceive | ยกเลิกรายการ |
| UserManagement | จัดการผู้ใช้ |
| PermissionManagement | จัดการสิทธิ์ |
| EmployeeManagement | จัดการพนักงาน |
| ProductTypeManagement | จัดการประเภทสินค้า |
| AppSettings | ตั้งค่าระบบ |

### การกำหนดสิทธิ์

1. คลิก **Permission Management**
2. เลือกผู้ใช้ที่ต้องการกำหนดสิทธิ์
3. ติ๊กเมนูที่ต้องการให้เข้าถึง
4. กด **Update Permission**
5. ผู้ใช้ต้อง **ออกจากระบบและเข้าใหม่** เพื่อให้สิทธิ์มีผล

## 20. การตั้งค่าระบบ (App Settings)

> **สิทธิ์**: ต้องมีสิทธิ์ App Settings จึงจะเข้าถึงได้

### การตั้งค่าที่ปรับได้

| การตั้งค่า | ค่าเริ่มต้น | คำอธิบาย |
|----------|----------|---------|
| Send Qty Tolerance | 2% | ค่าความคลาดเคลื่อนจำนวนส่ง |
| Store Qty Tolerance | 5% | ค่าความคลาดเคลื่อนจำนวนสต็อก |
| Bypass Mode | ปิด | เปิด/ปิดโหมด Bypass สำหรับทดสอบ |
| PIS API URL | (ตามสภาพแวดล้อม) | URL สำหรับเชื่อมต่อระบบ PIS |

### การแก้ไขการตั้งค่า

1. คลิก **App Settings**
2. แก้ไขค่าที่ต้องการ
3. กด **Update Settings**
4. ระบบบันทึกและใช้ค่าใหม่ทันที

> **คำเตือน**: การเปลี่ยนค่า Tolerance อาจส่งผลต่อการตรวจสอบจำนวนสินค้า โปรดระวังก่อนแก้ไข

## ภาคผนวก B: คำถามที่พบบ่อย (FAQ)

**Q: เข้าสู่ระบบไม่ได้ ทำอย่างไร?**
A: ติดต่อผู้ดูแลระบบเพื่อตรวจสอบบัญชีผู้ใช้ว่าถูกปิดการใช้งานหรือไม่

**Q: ทำไมไม่เห็นบางเมนู?**
A: เมนูที่แสดงขึ้นอยู่กับสิทธิ์ของผู้ใช้ ติดต่อผู้ดูแลระบบเพื่อขอสิทธิ์เพิ่มเติม

**Q: ยกเลิกการส่งสินค้าหลังจาก Confirm แล้วได้ไหม?**
A: ได้ โดยไปที่เมนู **Cancel Receive** และยกเลิกตาม Receive No หรือ Lot No

**Q: รายงานไม่แสดงหรือดาวน์โหลดไม่ได้?**
A: ตรวจสอบว่าเบราว์เซอร์ไม่ได้บล็อก Popup หรือ Download และลองใหม่อีกครั้ง

**Q: ค่า Tolerance 2% หมายความว่าอย่างไร?**
A: หากจำนวนสินค้าจริงต่างจากจำนวนที่อนุมัติไม่เกิน 2% ระบบจะถือว่าผ่าน ถ้าเกินจะต้องตรวจสอบเพิ่มเติม
